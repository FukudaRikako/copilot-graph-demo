Microsoft Graph の概要

Microsoft Graph とは

ふだん仕事で使っている
メール、予定表、ファイル、チャット——
これらは別々のアプリに見えますが、
裏側ではすべてつながっています。

その「つなぎ役」が
Microsoft Graph です。

たった一つの入口から、
Microsoft 365 のあらゆる情報を
まとめて取り出すことができます。
01 — ユーザー中心のデータモデル

「あなた」を中心に
すべてがつながる

Microsoft Graph はユーザーを中心にしたデータモデルです。

たとえば /me というエンドポイントから、
自分に関わるすべてのリソースにアクセスできます。

🔍 Graph エクスプローラー
マイクロソフト公式のツールで、
ブラウザ上から Graph API を実際に試すことができます。
developer.microsoft.com/en-us/graph/graph-explorer
📡 API ってなに？
API は「データを取得するための窓口」です。
ブラウザで URL を開くようなイメージで、
GET = 「このデータをください」 という命令を送ります。
※ GET は「取得する」という意味の英語です
💬 読み方：「GET /me」＝「私の情報をください」
GET /me　プロファイル取得
GET /me/messages　メール一覧
GET /me/events　予定表
GET /me/drive/root/children　ファイル
GET /me/joinedTeams　参加チーム
GET /me/people　関連する人

メール、予定、ファイル、同僚——
関係性で全部つながっているのが Graph の特徴です。
02 — Graph の仕組み

だれが、どうやって
データを届けるのか

Graph を使うとき、裏側では
3つの役割が連携しています。

🏢 Microsoft 365
　メールやファイルなどデータの保管庫

🔑 Microsoft Entra ID（旧 Azure AD）
　「この人は誰？」を確認する受付・身分証チェック

📡 Microsoft Graph
　身分証を見せれば何でも届けてくれる窓口

💡 レストランに例えると…
① Entra ID ＝ 入口の受付（予約確認）
② Graph ＝ ウェイター（注文を聞いて届ける）
③ Microsoft 365 ＝ 厨房（料理＝データを準備）

あなたは受付で身分を証明し（認証）、
ウェイターに「メールを見せて」と頼む（API）だけ。
厨房から料理（データ）が届きます。
03 — ユーザー体験層

☁️ Microsoft 365 コアサービス

あなたが毎日触れるサービス群です。
Graph API を通じてすべてプログラムからアクセスできます。

Teams Outlook Copilot Search Viva Engage
チャット、メール、会議、AI アシスタント、横断検索——
ユーザーが直接操作するインターフェースです。

💡 Graph でできること
• 次の会議の出席者のプロファイルを自動取得
• 予定表をスキャンして最適な会議時間を提案
• メールの重要度でフィルタリング

💬 API に送る命令の例（GET＝取得、POST＝送信）
GET /me/joinedTeams　← 参加チームを取得
GET /me/messages?$filter=...　← 重要メールだけ取得
POST /search/query　← 検索を実行
/me を起点に、体験層のデータへアクセス
04 — プラットフォーム層

⚡ データ基盤サービス

業務データが保存・管理される層です。
Graph はこれらを統一的に抽象化します。

SharePoint OneDrive Planner Excel OneNote To Do
ファイル、タスク、ノート、スプレッドシート——
業務データの実体がここにあります。

🔗 サービス間の関係性
• Teams のファイルは実は SharePoint に保存
• Planner のタスクは Groups に紐づく
• OneDrive から Excel API でリアルタイム操作

💬 API でデータを取得する例
GET /sites/{id}/drive　← サイトのファイル
GET /me/drive/root/children　← マイファイル一覧
GET /me/.../workbook　← Excel を操作
同じ GET 命令で、どのサービスのデータも取得可能
05 — 基盤・セキュリティ層

🔒 Enterprise Mobility + Security

ユーザーID・デバイスID・アクセス管理・コンプライアンス・
セキュリティを管理し、組織を保護する層です。

Entra ID 認証/OAuth Security Intune Reports
ID管理、脅威検出、デバイス管理、監査ログ——
全サービスの安全を守る基盤です。

🛡️ Graph のセキュリティ
• Graph API はセキュリティで保護されており
　承認された呼び出し元のみがアクセス可能
• OAuth 2.0 トークンによる認証・承認
• きめ細かなアクセス許可（スコープ）で制御

💬 セキュリティ情報を取得する API の例
GET /users/{id}　← ユーザー情報
GET /security/alerts_v2　← セキュリティ警告
GET /deviceManagement/...　← 管理デバイス一覧
組織を守る情報も、同じ GET 命令で取得できます
06 — まとめ

Microsoft Graph まとめ

📌 今日わかったこと
• Graph は M365 のあらゆるデータへの統一窓口
• /me を起点に、メール・予定・ファイル・チーム全部つながる
• Entra ID で認証し、GET ひとつでデータが届く

🌐 さらに広がる世界
Graph は Microsoft サービスだけでなく、
Copilot connectors（旧 Graph connectors）を使えば
Salesforce・ServiceNow・Jira などの
外部データも Graph に集約できます。
→ Copilot や Microsoft Search が社内外を横断検索

Graph があれば、
組織のあらゆる情報をつなぐことができる。

「あなたの組織では、
何と何をつなげたいですか？」

⚠️ ただし…
情報をつなぐためには、
「データの構造化」が大事なんです。

整理されていないデータは、
Graph でもうまくつなげません。

まずは自分たちのデータが
「つなげる状態か？」を考えることが第一歩です。
